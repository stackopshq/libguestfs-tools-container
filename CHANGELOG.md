## [1.0.0] - 2026-05-08

First release under the **StackOps** organization (`stackopshq/libguestfs-tools-container`). This is a full rewrite of the project — runtime, distribution, and packaging policies are all new. The previous incarnation under `Net-Architect-Cloud/docker-libguestfs-tools` is discontinued and not version-comparable.

### Changed
- **BREAKING**: Migrated container tooling from Docker to Podman/Buildah.
  - Renamed `Dockerfile` → `Containerfile` and `.dockerignore` → `.containerignore` (OCI-standard names; `podman build` consumes them natively).
  - Renamed `docker-compose.yml` → `compose.yml` (consumed by `podman-compose` and `docker compose v2`); removed obsolete `version: '3.8'` declaration.
  - All bind mounts in docs and example scripts now use the `:Z` SELinux relabel suffix.
  - Example scripts under `examples/` now use `podman run` and accept an `IMAGE` env var override.
- **BREAKING**: Renamed published image from `ghcr.io/net-architect-cloud/docker-libguestfs-tools` → `ghcr.io/stackopshq/libguestfs-tools`. Source repository moved from `Net-Architect-Cloud/docker-libguestfs-tools` to `stackopshq/libguestfs-tools-container`. Existing pull commands must be updated.
- **BREAKING**: Image now published in **two variants**:
  - `:latest`/`:slim` (default) — drops `virt-v2v` and `ntfs-3g`. Targets Linux VM image manipulation (virt-customize/virt-sysprep/virt-sparsify), which is the openimages.cloud build-pipeline use case.
  - `:full` — adds `virt-v2v` and `ntfs-3g` for VMware/Hyper-V → KVM conversions and Windows guest support.
  Pull `:full` if you previously relied on virt-v2v from `:latest`.
- **BREAKING**: Base image changed from `quay.io/rockylinux/rockylinux:10-ubi-init` → `quay.io/rockylinux/rockylinux:10-ubi`. The libguestfs appliance boots its own kernel/init in a child VM, so the host container does not need systemd-as-pid1.
- **BREAKING**: Replaced `qemu-kvm` (meta-package) with `qemu-kvm-core`. Drops `qemu-kvm-ui-opengl` → `mesa-dri-drivers` → `llvm-libs` (~165 MB). The `direct` libguestfs backend launches qemu in `-display none`, so graphical/audio sub-packages are unused.
- Image size reduced from ~1061 MB → **~647 MB for `:slim`** (-39%) and ~770 MB for `:full` after removing transitive bloat (locales, doc dirs, build IDs, and unused utilities `git`/`wget`/`procps-ng`/`iproute`).
- CI workflow rewritten on top of `redhat-actions/buildah-build` / `podman-login` / `push-to-registry`. Each `(arch × variant)` combination runs on a **native** GitHub-hosted runner: `ubuntu-latest` for amd64, `ubuntu-24.04-arm` for arm64 — replaces the previous QEMU-emulated multi-arch path (`qemu-user-static`). The build matrix now produces 4 per-arch images that a dedicated `manifest` job assembles into manifest lists. The smoke test runs `qemu-img --version`, `virt-inspector --version`, and (on `:full`) `virt-v2v --version`; deeper appliance testing via `libguestfs-test-tool` is left to the **downstream openimages.cloud workflows**, which run on self-hosted Rocky 10 runners with nested virtualization (`/dev/kvm`).

### Added
- **Cosign keyless signing** (Sigstore, GitHub OIDC) on every published image. Verify with:
  ```bash
  cosign verify ghcr.io/stackopshq/libguestfs-tools:latest \
    --certificate-identity-regexp '^https://github.com/stackopshq/libguestfs-tools-container/' \
    --certificate-oidc-issuer https://token.actions.githubusercontent.com
  ```
- **SPDX SBOM** generated by `anchore/sbom-action` and attached as a cosign attestation (predicate type `spdxjson`). Also uploaded as a workflow artifact (90-day retention).
- **OCI provenance labels** populated from CI build args: `org.opencontainers.image.version`, `revision`, `created`, plus a custom `cloud.openimages.variant` label identifying `slim`/`full`.
- Multi-arch manifest lists assembled in a dedicated `manifest` job from per-arch builds (`-amd64`/`-arm64`), allowing native arm64 builds on dedicated arm64 runners.
- Strip pass on `/usr/share/locale/*` (kept only `C`, `C.utf8`, `en`, `en_US`) and removal of `/usr/lib/.build-id`, `/root/.cache`.

### Removed
- `git`, `wget`, `procps-ng`, `iproute` from the default install — never used in the openimages.cloud build pipeline; runners clone repos themselves and `curl` covers the wget use cases.
- `dnf -y update` from the build (the Rocky base image is already current at publication; re-updating only invalidates caches and inflates the layer).
- `VOLUME ["/workspace/shared"]` directive — anti-pattern that creates anonymous volumes per `podman run` without matching `-v`. Volume contracts now belong to the caller.
- `qemu-user-static` step from CI (no longer needed: arm64 builds run natively on arm64 runners).
